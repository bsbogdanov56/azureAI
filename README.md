# Azure — буфер-ассистент по горячей клавише

Маленькая утилита для Windows: по `Ctrl+Shift+Q` берёт текст из буфера обмена, отправляет его в LLM и печатает ответ туда, где стоит курсор. История диалога сохраняется между запусками.

## Как работает

1. Скопировал текст (`Ctrl+C`).
2. Нажал `Ctrl+Shift+Q`.
3. Программа печатает `...`, шлёт запрос, заменяет `...` на ответ модели.
4. Ответ заодно кладётся в буфер обмена — можно сразу вставить куда угодно.

При ошибке запроса вместо ответа печатается `!`, а текст ошибки попадает в буфер.

## Стек

- Python 3.10+
- [`keyboard`](https://pypi.org/project/keyboard/) — глобальный хоткей и эмуляция ввода
- [`pyperclip`](https://pypi.org/project/pyperclip/) — буфер обмена
- [`requests`](https://pypi.org/project/requests/) — HTTP-клиент
- API: OpenAI-совместимый эндпоинт (по умолчанию `routerai.ru`, модель `deepseek/deepseek-v3.2`)

## Установка

```bash
git clone <repo>
cd <repo>
pip install -r requirements.txt
```

На Windows для глобальных хоткеев `keyboard` лучше запускать **с правами администратора** — иначе перехват клавиш в некоторых приложениях не сработает.

## Настройка

Открой `azure.py` и впиши свой ключ в строку:

```python
API_KEY = ""   # ← вставь сюда ключ от провайдера
```

По умолчанию используется [routerai.ru](https://routerai.ru) с моделью `deepseek/deepseek-v3.2`. Если хочешь другого провайдера (OpenAI, OpenRouter и т.п.) — поменяй ещё `API_URL` и `MODEL`, эндпоинт должен быть OpenAI-совместимым (`/v1/chat/completions`).

Дополнительно правится:
- `MAX_TURNS` — глубина истории (по умолчанию 12 пар user/assistant)
- хоткей в строке `keyboard.add_hotkey("ctrl+shift+q", ...)`

## Запуск

```bash
python azure.py
```

Окно консоли можно свернуть — программа висит фоном и слушает хоткей. Завершить — `Ctrl+C` в консоли.

## Сборка в exe

```bash
pip install pyinstaller
pyinstaller --onefile --noconsole azure.py
```

Готовый `azure.exe` появится в `dist/`.

## Файлы

- `azure.py` — основной скрипт
- `chat_history.json` — создаётся автоматически рядом со скриптом, хранит контекст диалога
- `requirements.txt` — зависимости

## Особенности

- Перед отправкой удаляется markdown-форматирование из ответа (модель часто сыплет `**` и `*`, что некрасиво смотрится при вставке в обычный текст).
- История обрезается до `MAX_TURNS` последних обменов, system-prompt сохраняется.
- `SESSION.trust_env = False` отключает чтение системных прокси — полезно за корпоративным VPN.
